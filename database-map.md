# Database Schema & Entity Map — Spend Analysis

This document details the MongoDB database architecture managed via Mongoose 9 inside [db.js](file:///c:/Antigravity/expense/manager/backend/db.js).

---

## 1. Mongoose Collections Map

### `Analysis` Collection
Stores metadata and redacted transaction payloads.

* **Purpose**: Persists historical analysis summaries for admin tracking, while storing redacted statements for session persistence.
* **Fields**:
  * `id` (`String`, required, unique): UUID v4 lookup key.
  * `period` (`String`): Date range for the statement (e.g., `"01 Apr – 25 Apr 2026"`).
  * `bank` (`String`): Name of the detected bank (e.g., `"HDFC Bank"`).
  * `account_holder` (`String`): Masked or redacted account holder name.
  * `total_spent` (`Number`): Sum of all debit transactions (excluding Self Transfers).
  * `transaction_count` (`Number`): Total count of transactions parsed.
  * `is_redacted` (`Boolean`): Flag indicating if PII redaction has been applied (`true`).
  * `data` (`Mixed`): Nested document containing the redacted transactions and insights payload.
  * `created_at` (`Date`, default: `Date.now`): Ingestion timestamp.
* **Indexes**: Unique index on `id`.

---

### `AuditLog` Collection
Stores system logs for admin monitoring.

* **Purpose**: Tracks operational history and security events.
* **Fields**:
  * `id` (`String`, required): Unique identifier (Epoch timestamp + random index).
  * `timestamp` (`Date`, default: `Date.now`): Log creation timestamp.
  * `action` (`String`): Action type.
    * Values: `ADMIN_LOGIN`, `ADMIN_LOGIN_FAILED`, `PASSWORD_CHANGED`, `ANALYSIS_CREATED`, `ANALYSIS_FAILED`, `ANALYSIS_VIEWED`, `ANALYSIS_UPDATED`, `ANALYSIS_DELETED`, `CSV_EXPORTED`.
  * `details` (`String`): Detailed text log.
  * `ip` (`String`): Hashed or masked client IP address.

---

### `ApiUsage` Collection
Stores daily API performance metrics.

* **Purpose**: Monitors Gemini API latency and token consumption.
* **Fields**:
  * `date` (`String`, required, unique): Calendar date (`YYYY-MM-DD`).
  * `provider` (`String`, default: `"gemini"`): AI provider name.
  * `total_calls` (`Number`, default: 0): Total requests sent.
  * `successful_calls` (`Number`, default: 0): Successful responses.
  * `failed_calls` (`Number`, default: 0): Failed requests.
  * `total_tokens_estimated` (`Number`, default: 0): Total estimated token usage.
  * `latencies` (`[Number]`, default: `[]`): Last 100 API request latencies (ms).
  * `errors` (`[Mixed]`, default: `[]`): Last 50 error details (timestamp + message).
* **Indexes**: Unique index on `date`.

## 2. Document Hierarchies & Nested Entities

MongoDB uses nested documents rather than relational tables. Below is the document hierarchy:

### Entity Relationship Tree

```
┌────────────────────────────────────────────────────────┐
│                   Analysis Document                    │
└────────────────────────────────────────────────────────┘
                           │
             ┌─────────────┴─────────────┐
             ▼                           ▼
 ┌───────────────────────┐   ┌───────────────────────┐
 │  Transactions Array   │   │    Insights Array     │
 └───────────────────────┘   └───────────────────────┘
   * date                      * icon
   * desc                      * title
   * amount                    * body
   * cat                       * badge
   * reward_points             * color
```

* **Analysis (Parent) $\to$ Transactions (Child)**: One-to-many embedded relationship. Category corrections updated in the UI are synced directly to this array.
* **Analysis (Parent) $\to$ Insights (Child)**: One-to-many embedded relationship. Houses natural language recommendations generated by Gemini.
